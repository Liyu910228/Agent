# 多 Agent 用户问题处理平台 PRD

## 1. 产品目标

搭建一个自动化多 Agent 问题处理平台。业务员只需要输入用户问题，系统自动识别问题类型，选择合适 Agent、模型、Skill 和 MCP 工具协作处理，并返回最终答案。管理员负责配置和查看 Agent、模型、Skill、MCP 工具以及处理记录。

## 2. 用户角色

### 管理员

- 账号：admin
- 密码：admin
- 进入管理员页面
- 可查看和配置 Agent、模型、Skill、MCP 工具
- 可查看所有问题处理记录和调用链

### 业务员

- 账号：root
- 密码：12345
- 进入业务员页面
- 可提交用户问题
- 可查看最终答案、处理状态和调用链

## 3. 页面范围

### 登录页

- 输入账号和密码
- 根据角色跳转到对应页面
- 错误账号或密码显示失败提示

### 管理员页面

- 总览指标
- Agent 列表和配置
- 模型列表和刷新
- Skill 列表
- MCP 工具列表
- 问题处理记录
- 调用链详情

### 业务员页面

- 用户问题输入
- 提交问题
- 处理状态展示
- 最终答案展示
- 参与 Agent、模型、Skill、MCP 工具展示
- New chat 按钮清空当前聊天记录
- 输入 `/` 打开能力菜单，可直接指定本次调用的 Skill 或 MCP

## 4. 核心功能

### 登录与角色跳转

第一版使用本地账号，不接真实用户系统。

- admin/admin 跳转管理员页面
- root/12345 跳转业务员页面

### 自动多 Agent 调度

业务员不手动选择流程。系统根据用户问题自动判断问题类型并选择 Agent。

内置 Agent：

- Router Agent：判断问题类型和处理策略
- General Agent：处理普通问答
- Research Agent：处理资料、政策、知识类问题
- Reasoning Agent：处理复杂分析、方案、建议类问题
- Tool Agent：处理查询、订单、业务操作类问题
- Reviewer Agent：检查答案质量并整理最终回复

### 真实模型 API 接入

后端读取环境变量：

- LLM_BASE_URL
- LLM_API_KEY
- LLM_DEFAULT_MODEL

前端不能直接接触 API Key。仓库只提交 `.env.example`，不提交 `.env`。

### 模型列表获取

系统通过模型服务地址和 Key 获取可用模型列表，并在管理员页面展示。

### 自动模型选择

系统根据任务类型选择模型：

- 普通问题：优先速度快、成本低的模型
- 复杂分析：优先推理能力强的模型
- 工具调用：优先支持 tool/function calling 的模型
- 长文本：优先长上下文模型
- 审核总结：优先稳定、低成本模型

如果无法判断，使用默认模型。

### Skill 自动调用

系统内置 Skill 注册表，Router Agent 根据问题自动选择是否调用 Skill。

第一版内置 Skill：

- classify_question
- summarize_text
- analyze_problem
- extract_action_items

### MCP 自动调用

第一版实现 MCP 工具抽象层，预留真实 MCP Client 接入。

第一版内置模拟 MCP 工具：

- search_docs
- query_order_status
- create_ticket

### 处理记录与调用链

每次问题处理记录：

- 用户问题
- 问题类型
- 参与 Agent
- 使用模型
- 调用 Skill
- 调用 MCP 工具
- 每一步输入输出
- 耗时
- 成功或失败状态
- 错误信息
- 最终答案

## 5. 自动路由规则

- 包含“怎么、为什么、分析、方案、建议”等：Reasoning Agent
- 包含“查询、订单、状态、编号、客户”等：Tool Agent
- 包含“资料、文档、政策、知识库、规定”等：Research Agent
- 其他简单问题：General Agent
- 非简单问题默认经过 Reviewer Agent

## 6. 第一版不做

- 用户自定义流程编排
- 拖拽式工作流
- 多租户
- 计费
- Agent 市场
- 复杂人工审批流
- 真实企业权限体系
- 大规模分布式任务调度
- 模型微调
- 完整向量知识库

## 7. 验收标准

### 登录

- 输入 admin/admin 后进入管理员页面
- 输入 root/12345 后进入业务员页面
- 错误账号或密码显示登录失败提示

### 管理员页面

- 能看到内置 Agent 列表
- 能看到模型列表和刷新结果
- 能看到 Skill 和 MCP 工具列表
- 能看到问题处理记录
- 能查看每条记录的调用链和最终答案

### 业务员页面

- 能输入用户问题并提交
- 能看到处理中状态
- 能看到最终答案
- 能看到参与 Agent、模型、Skill、MCP 工具
- 能点击 New chat 清空当前页面聊天记录
- 能通过 `/` 菜单选择本次问题要直接调用的 Skill 或 MCP

### 模型接入

- 后端能通过 LLM_BASE_URL 和 LLM_API_KEY 获取模型列表
- 获取失败时返回明确错误
- 每条处理记录能看到实际使用模型
- API Key 不出现在前端源码、页面或日志中

### Skill 和 MCP

- 系统能根据问题自动选择 Skill 或 MCP 工具
- 调用结果能进入最终答案生成流程
- 调用详情能出现在处理记录中
